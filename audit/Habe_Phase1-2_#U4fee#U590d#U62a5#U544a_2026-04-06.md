## 已完成的修复

### Phase 1.1: ✅ 实现 `onCharacteristicWrite` 回调处理

**文件：** `app/src/main/java/com/shawnrain/habe/ble/BleManager.kt`

**修改内容：**
1. 添加 `WriteResult` 密封类，表示写入结果（成功/丢弃/失败）
2. 添加 `_writeResults` SharedFlow 用于向调用者传播写入结果
3. 实现 `onCharacteristicWrite` 回调（兼容 API 33 前后）
4. 在回调中记录状态码并通过 `_writeResults` 发射结果

**影响：**
- 现在可以通过监听 `bleManager.writeResults` 获取每次 BLE 写入的结果
- 智科参数写入不再"黑盒"操作，失败时可追溯

### Phase 1.2: ✅ 修复 `writeBytes` 静默丢弃问题

**文件：** `app/src/main/java/com/shawnrain/habe/ble/BleManager.kt`

**修改内容：**
1. `writeBytes` 返回类型从 `Unit` 改为 `WriteResult`
2. characteristic 为 null 时：
   - 记录警告日志
   - 发射 `WriteResult.Dropped` 到 `_writeResults`
   - 返回明确结果对象
3. 添加 try-catch 包裹写入操作，捕获异常时发射 `WriteResult.Failed`

**影响：**
- 调用者现在可以检查返回值或监听 `writeResults` 流获取写入状态
- 轮询循环写入丢弃时会记录日志，便于调试"数据全 0"问题

---

### Phase 1.3: ✅ 为 `startScan()` 添加超时、过滤器和权限检查

**文件：** `app/src/main/java/com/shawnrain/habe/ble/BleManager.kt`

**修改内容：**
1. **权限检查：**
   - 添加 `hasBluetoothScanPermission()` 函数
   - Android 12+ 检查 `BLUETOOTH_SCAN`
   - Android 11 及以下检查 `ACCESS_FINE_LOCATION`
   - 扫描前检查权限，无权限时记录警告并返回

2. **扫描过滤器：**
   - 添加 Service UUID 过滤器（`0000FFE0`，智科控制器服务）
   - 使用 `ScanSettings.SCAN_MODE_LOW_LATENCY` 提高扫描优先级

3. **超时机制：**
   - 添加 `SCAN_TIMEOUT_MS = 10_000L`（10 秒）
   - 使用 `scanTimeoutRunnable` 自动停止扫描
   - `stopScan()` 时移除超时回调

4. **扫描失败回调：**
   - 实现 `onScanFailed(errorCode)` 记录错误并清空设备列表

5. **竞态条件修复：**
   - `connect()` 前调用 `stopScan()` 停止活跃扫描
   - 不再无条件覆盖 `pendingDeviceNameHint`

**影响：**
- 扫描不会无限继续，10 秒后自动停止
- 过滤非目标设备，减少回调噪声
- Android 12+ 权限合规

### Phase 1.4: ✅ 处理 `onServicesDiscovered` 失败路径

**文件：** `app/src/main/java/com/shawnrain/habe/ble/BleManager.kt`

**修改内容：**
1. 添加 `else` 分支处理 `status != GATT_SUCCESS`
2. 记录错误日志包含设备地址和状态码
3. 将连接状态转换为 `ConnectionState.Error("服务发现失败: status=$status")`
4. 调用 `gatt.disconnect()` 允许重试

**影响：**
- 服务发现失败不再"卡死"在 Connected 状态
- UI 可监听 `ConnectionState.Error` 显示错误信息
- 自动断开连接，允许看门狗扫描重新连接

### Phase 1.5: ✅ 在 `MainViewModel.onCleared()` 中添加 BLE 断开连接逻辑

**文件：** `app/src/main/java/com/shawnrain/habe/MainViewModel.kt`

**修改内容：**
```kotlin
override fun onCleared() {
    AppLogger.i(TAG, "MainViewModel 被清理，清理 BLE 资源")
    // Disconnect BLE managers to prevent lingering connections
    bleManager.disconnect()
    bmsBleManager.disconnect()
    runCatching { kotlinx.coroutines.runBlocking { lanBackupTransfer.stop() } }
    headingTracker.stop()
    gpsTracker.stopTracking()
    super.onCleared()
}
```

**影响：**
- ViewModel 销毁时主动断开 BLE 连接
- 防止 GATT 对象泄漏和外设连接残留
